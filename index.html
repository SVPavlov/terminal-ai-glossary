Reference Guide

Terminal & AI Glossary

A practical reference for non-developers working with Claude Code, AI agents, and modern developer tools. Plain language. No jargon chains.

💻

Shell & Terminal Basics

Terminal
The application window where you type commands. Think of it as the screen and keyboard for talking directly to your computer. Examples: iTerm2, Ghostty, Warp, WezTerm.
Shell
The program running inside the terminal that interprets your commands. The terminal is the window; the shell is the brain behind it. Bash and Zsh are the two most common shells.
Bash
"Bourne Again SHell" — the default shell on most Linux systems and older Macs. The most widely documented shell. If you open a terminal on most servers, you’re using Bash.
Zsh
"Z Shell" — the default shell on modern macOS. Works like Bash but with better auto-completion, customization, and convenience features.
CLI
Command Line Interface. Any tool you interact with by typing text commands rather than clicking buttons. Claude Code is a CLI tool.
GUI
Graphical User Interface. The point-and-click alternative to CLI. VS Code is a GUI; the terminal is a CLI.
Console
Often used interchangeably with "terminal." Technically the physical screen + keyboard, but in practice people mean the same thing.
TTY
"Teletypewriter" — a historical term for terminal connections. You might see it in error messages. It basically means "a terminal session."
Prompt
The blinking cursor area in your terminal where you type commands (e.g., user@server:~$). Not to be confused with an AI prompt.
Command
A single text instruction you type in the terminal, like ls (list files) or cd (change directory).
Flag / Option
A modifier added to a command with a dash. In git commit -m "message", the -m is a flag that says "here comes my message."
Argument
The value you pass to a command. In cd Documents, "Documents" is the argument.
Script
A text file containing a sequence of commands that run automatically, like a recipe for the shell. Usually ends in .sh.
REPL
"Read-Eval-Print Loop" — an interactive environment where you type a command, it runs immediately, and you see the result. Python's interactive shell is a REPL.
Stdin / Stdout / Stderr
The three channels every command uses: stdin for receiving input, stdout for normal output, stderr for error messages. You usually see stdout and stderr in your terminal, but they can be separated.
Exit Code
A number a command returns when it finishes. 0 means success, anything else means something went wrong. Scripts use these to know if a step worked.
Tab Completion
Pressing the Tab key to auto-complete file names, commands, or options. Saves typing and prevents typos.
Man Page
The built-in manual for a command. Type man git to read git's manual. Press q to quit.
Alias
A shortcut you create for a longer command. Example: alias gs="git status" so you only type gs.
🔧

Terminal Tools & Multiplexers

tmux
Terminal MUltipleXer. Lets you split your terminal into multiple panes, keep sessions alive when you disconnect, and run multiple things at once. Essential for VPS work.
Zellij
A modern alternative to tmux with a friendlier interface. Easier to learn but less compatible with tools like claude-squad.
Warp
A modern terminal with built-in AI features. Beginner-friendly but requires account creation.
Ghostty
A fast, GPU-accelerated terminal by HashiCorp's founder. Recommended as the best Mac terminal in 2026.
WezTerm
Cross-platform terminal emulator. A solid choice for Windows users who want a modern terminal experience.
Session
A tmux/terminal session that persists even when you close the window or disconnect. You can "attach" back to it later — perfect for long-running tasks on a VPS.
Pane
A subdivision of a terminal window. In tmux, you can split one window into multiple panes, each running different commands side by side.
Split
Dividing your terminal into two or more visible areas. Horizontal split = top/bottom. Vertical split = left/right.

Infrastructure & Hosting

VPS
Virtual Private Server. A remote computer you rent to run your tools 24/7. Providers like DigitalOcean and Hetzner offer them from ~$18/month.
SSH
Secure Shell. The encrypted protocol to remotely connect to another computer's terminal over the internet. ssh user@server-ip connects you.
SSH Key
A pair of cryptographic files (public + private) used instead of a password to log into servers. More secure and convenient than typing passwords.
Server
A computer that runs continuously, usually without a screen. Can be a VPS in the cloud or an old machine at home.
Cloud
Someone else's servers that you rent on demand. AWS, Google Cloud, Azure, DigitalOcean, Hetzner are cloud providers.
Docker
A tool that packages software into isolated "containers" so it runs the same everywhere. Think of it as a lightweight virtual computer inside your computer.
Container
A lightweight, isolated environment created by Docker. Runs a program in a sandbox so it cannot access the rest of your computer. Running Claude Code in a container = safer.
Docker Compose
A tool to define and run multiple Docker containers together using a single docker-compose.yml config file.
Image
A snapshot/template from which Docker containers are created. You download an image once, then run as many containers from it as you need.
Port
A numbered "door" on a computer where a specific service listens. Port 80 = web traffic, port 22 = SSH, port 3000 = often a dev server.
Localhost
Your own computer, accessed via 127.0.0.1 or the name localhost. When a tool says it's running on localhost:3000, it means "on this machine, port 3000."
Domain
The human-readable address of a website (e.g., prowareness.academy). A DNS server translates it to a numeric IP address.
DNS
Domain Name System. The internet's phone book — translates domain names like google.com into IP addresses.
IP Address
A numeric address identifying a computer on a network (e.g., 192.168.1.1). Every VPS has one.
Firewall
Software that controls which connections are allowed in and out of a server. Essential security for any VPS.
Reverse Proxy
A server (like Nginx or Caddy) that sits in front of your application and forwards requests to it. Handles SSL, load balancing, and routing.
Nginx
A popular web server and reverse proxy. Often used to put a public-facing HTTPS layer in front of applications running on a VPS.
NAS
Network Attached Storage. A dedicated box (like Synology) for storing files on your home network.
Tailscale
A zero-config VPN that connects your devices securely. Used for accessing Claude Code from your phone without exposing it to the internet.
Systemd
The system manager on most Linux servers. Controls which services start on boot and keeps them running. Interact with it via systemctl.
Daemon
A background process that runs continuously on a server. Your web server, database, and WhatsApp bridge are all daemons.
SCP
Secure Copy. Transfer files between your computer and a remote server over SSH. scp file.txt user@server:/path/
UPS
Uninterruptible Power Supply. A battery backup for home servers so they power down gracefully during power outages.

Git & Version Control

Git
The most widely used version control system. Tracks every change you make to files, so you can go back in time. Like an infinite undo history that also lets multiple people collaborate.
Repository (repo)
A project folder tracked by Git. Contains your files plus the full history of every change. Your Second Brain lives in a repo.
Clone
Downloading a copy of a repository from GitHub to your local machine for the first time.
Commit
A saved snapshot of your changes with a description of what you did. Like a checkpoint in a video game.
Stage (git add)
Marking specific changed files to be included in your next commit. You stage files, then commit them.
Push
Uploading your local commits to a remote repository (like GitHub) so others can see them.
Pull
Downloading the latest changes from a remote repository to your local machine.
Branch
A parallel version of your code. You create a branch to try something without affecting the main version. Like making a copy of a document to experiment on.
Main / Master
The primary branch of your repository. main is the modern convention; master is the older term.
Merge
Combining changes from one branch back into another. When your experiment works, you merge it into main.
Pull Request (PR)
A proposal to merge changes from one branch into another. Others can review your changes, comment, and approve. GitLab calls it "Merge Request."
Conflict
When Git cannot automatically combine changes because two people edited the same lines. You need to manually choose which version to keep.
Diff
A comparison showing exactly what changed between two versions of a file. Added lines show in green, removed lines in red.
Remote
The version of your repository hosted online (usually on GitHub). Your local repo "talks to" the remote via push and pull.
Origin
The default name for the remote you cloned from. git push origin main pushes to the original source.
Fork
Your personal copy of someone else's repository on GitHub. You fork, make changes, then submit a pull request back to the original.
Checkout
Switching between branches or restoring files. Boris (Claude Code creator) uses "5-10 separate git checkouts of the same repo" for parallel agents.
Worktree
A Git feature that lets you have multiple branches checked out simultaneously in different folders. Used by claude-squad and dmux for parallel AI agents.
Stash
Temporarily saving uncommitted changes so you can switch branches. Like putting papers in a drawer before clearing your desk. git stash saves; git stash pop restores.
HEAD
A pointer to the commit you are currently working on. Usually points to the latest commit on your current branch.
Tag
A named marker on a specific commit, usually used for release versions (e.g., v1.0.0).
.gitignore
A file that tells Git which files to NOT track. Used for passwords, large files, or temporary files you do not want in your repo.
Git Hooks
Scripts that run automatically before or after Git operations. Can block destructive commands to prevent AI agents from accidentally deleting work.
gh (GitHub CLI)
A command-line tool for interacting with GitHub. Create PRs, view issues, manage repos without opening a browser.

Package Management

Package
A bundle of code someone else wrote that you can install and use. Like an app from an app store, but for code.
Package Manager
A tool that installs, updates, and removes packages. Like an app store for the command line.
npm
Node Package Manager. Installs JavaScript/Node.js packages. You'll see npm install in many setup instructions.
npx
Runs an npm package without permanently installing it. Useful for one-off commands.
pip
Python's package manager. pip install duckdb installs the DuckDB database tool.
Homebrew (brew)
The most popular package manager for macOS. brew install ghostty installs the Ghostty terminal.
apt
The package manager on Ubuntu/Debian Linux. apt install nginx installs the Nginx web server. Your VPS likely uses this.
Dependency
A package your project needs to work. If tool A requires tool B, B is a dependency of A.
node_modules
The folder where npm stores downloaded packages. Can get very large. Never commit it to Git.
package.json
A file listing all npm dependencies for a project. Like a shopping list of required packages.
package-lock.json
Auto-generated file recording the exact version of every package installed. Ensures everyone gets identical packages.
Semver
Semantic Versioning. A numbering scheme (e.g., 2.1.3) where first = breaking changes, second = new features, third = bug fixes.
Registry
The online catalog where packages are stored. npmjs.com for npm, PyPI for Python.
Virtual Environment (venv)
A Python feature creating an isolated space for a project's packages, so different projects don't interfere with each other.

AI / LLM Developer Terms

LLM
Large Language Model. The AI technology behind Claude, ChatGPT, Gemini. Trained on vast amounts of text to understand and generate language.
Token
The unit LLMs use to process text. Roughly 3/4 of a word. "Scrum Master" is about 3 tokens. Pricing and limits are measured in tokens.
Context Window
The total amount of text an LLM can "see" at once. Claude's context window is up to 200K tokens (about 500 pages). Your input plus its output combined.
Prompt
The instruction you give to an AI. In Claude Code, everything you type is a prompt. Good prompting = good results.
System Prompt
Hidden instructions given to the AI before your conversation starts. Defines personality, rules, and behavior. CLAUDE.md acts like a system prompt for Claude Code.
Context Engineering
Designing systems that give AI the right information at the right time. Not just prompting, but architecting how context flows through your tools.
Agent
An AI that can take actions (run commands, edit files, search the web) rather than just generate text. Claude Code operates as an agent.
Agentic Workflow
A process where an AI agent performs multiple steps independently: reading files, running commands, making decisions, and producing results.
Agent Swarm / Multi-Agent
Running multiple AI agents simultaneously on different tasks. Example: "Codex creates the plan, Claude Code builds, Codex reviews work."
MCP
Model Context Protocol. An open standard that lets AI assistants connect to external tools and data sources. Like USB ports for AI — plug in Gmail, Notion, Supabase, etc.
MCP Server
A small program that exposes a specific tool or data source to AI via the MCP protocol. The Notion MCP server lets Claude read/write Notion pages.
MCP Client
The AI application (like Claude Code or Claude Desktop) that connects to MCP servers to use their tools.
Tool Use / Function Calling
The ability of an AI to invoke external tools (search the web, read files, run code) as part of generating a response.
Skill
A reusable instruction set for Claude Code. Like a saved workflow you can trigger with a slash command. Example: /plan, /review, /morning.
Hook
Code that runs automatically when certain events happen in Claude Code. Pre-tool hooks can block dangerous operations. Post-tool hooks can log actions.
CLAUDE.md
The configuration file that tells Claude Code about your project, preferences, and rules. The brain's instruction manual.
RAG
Retrieval-Augmented Generation. The AI searches your documents first, then uses the results to answer — grounding responses in your actual data.
Embedding
Converting text into numbers (vectors) that capture meaning. "dog" and "puppy" have similar embeddings. Powers semantic search in tools like LanceDB.
Vector Database
A specialized database that stores embeddings and can quickly find the most similar items. Used to give AI systems long-term memory.
Hallucination
When an AI generates confident-sounding information that is factually incorrect. Always verify critical facts.
Grounding
Techniques to connect AI responses to real data (documents, databases, search results) to reduce hallucinations.
Guardrails
Rules and constraints placed on an AI system to keep its behavior safe, accurate, and on-topic.
Temperature
A setting controlling how creative/random AI responses are. Low = more predictable and focused. High = more varied and creative.
Inference
The process of the AI generating a response to your prompt. When Claude is "thinking," it is performing inference.
Fine-tuning
Training a pre-existing AI model on your specific data to specialize it. Claude Code's context engineering achieves similar results without fine-tuning.
Multimodal
An AI that can process multiple types of input — text, images, PDFs, code — not just text. Claude is multimodal.
Streaming
When AI output appears word-by-word in real time instead of all at once. The typing effect you see in Claude Code.
Model ID
The specific identifier for an AI model version, like claude-opus-4-6. Different IDs = different capabilities and costs.
Knowledge Cutoff
The date up to which an AI was trained. It may not know about events after this date unless it can search the web.
API
Application Programming Interface. A structured way for programs to communicate. When Claude Code calls GitHub, it's sending structured requests to GitHub's API.
API Key
A secret password that identifies you when calling an API. Never put these in code files — use environment variables or keychains.
OAuth
A standard for letting applications access your accounts (Google, Microsoft) without sharing your password. Used by MCP servers to connect to services.
Prompt Injection
A security attack where malicious text tricks an AI into ignoring its instructions. A key concern for MCP-connected systems.
Token Budget
The amount of context window space available for a conversation. MCPs, skills, and CLAUDE.md all consume tokens. Tip: fewer MCPs = more room for actual work.
Compact / /compact
A Claude Code command that summarizes the current conversation to free up context window space. Use when sessions get long.

CLI Commands & Concepts

cd
Change Directory. Navigate to a different folder. cd ~/Documents goes to Documents. cd .. moves up one level.
ls
Lists files and folders. ls -la shows all files including hidden ones with details.
cat
Displays the contents of a file. cat README.md prints the file to screen. Short for "concatenate."
grep
Search for text patterns inside files. grep "MCP" CLAUDE.md finds all lines mentioning "MCP."
curl
Make web requests from the terminal. Used to test APIs. curl https://api.example.com fetches data from a URL.
wget
Download files from the web. wget https://example.com/file.zip saves a file to your current folder.
mkdir
Make directory. mkdir new-project creates a new folder.
rm
Remove (delete) files. rm file.txt deletes a file. rm -rf folder/ deletes a folder. Be careful — no recycle bin.
cp / mv
Copy and move files. cp file.txt backup.txt copies; mv file.txt new-name.txt renames.
touch
Creates an empty file or updates a file's timestamp. touch notes.txt creates a new empty file.
echo
Prints text to the screen. echo "Hello" displays "Hello." Also used to check variables: echo $PATH.
which
Shows where a command lives on your system. which node tells you the path to the Node.js executable.
tail
Shows the last lines of a file. tail -f log.txt follows a log file in real time — very useful for debugging.
pwd
Print Working Directory. Shows the full path of the folder you're in.
Pipe ( | )
Sends the output of one command as input to another. cat file.txt | grep "error" reads a file then searches for "error."
Redirect ( > , >> )
Sends command output to a file. > overwrites; >> appends. Example: echo "hello" > file.txt
Environment Variable
A named value stored in the shell's memory. Used for configuration and secrets. API_KEY=abc123 sets one. Access with $API_KEY.
PATH
A special environment variable listing directories where the shell looks for commands. If "command not found" — it's probably not in your PATH.
Dotfiles
Configuration files starting with a dot (.bashrc, .gitconfig). Hidden by default. They customize how your tools behave.
.bashrc / .zshrc
The startup config file for Bash or Zsh. Runs every time you open a terminal. Used for aliases, PATH, and environment variables.
.env
A file containing environment variables for a project (API keys, database URLs). Should never be committed to git.
sudo
"Super User DO." Runs a command with administrator privileges. Use with caution — it can modify system files.
chmod
Change Mode. Sets file permissions. chmod +x script.sh makes a script executable.
Process
A running instance of a program. Each command creates a process. ps lists them; kill stops them.
Background Process ( & )
Adding & at the end of a command runs it in the background so you can keep using the terminal.
Ctrl+C
Keyboard shortcut to stop (kill) a running command. Your go-to "make it stop" key combination.
Ctrl+D
Signals "end of input." Closes interactive sessions and logs you out of SSH.
Ctrl+Z
Pauses (suspends) a running command. Use fg to resume in foreground or bg to resume in background.
Wildcard ( * )
A pattern that matches anything. *.md means "all .md files." **/*.md means "all .md files in any subfolder."
Home Directory ( ~ )
Your user's personal folder. On Mac: /Users/yourname. The ~ symbol is a shortcut for it.
sqlite3
Command-line tool for SQLite databases. Used to query local databases like the WhatsApp bridge database.

DevOps & Automation

CI/CD
Continuous Integration / Continuous Deployment. Automated pipelines that test code when you push it (CI) and deploy it automatically if tests pass (CD).
GitHub Actions
GitHub's built-in CI/CD system. Defined in YAML files in .github/workflows/. Runs tests, builds, and deploys on every push.
Workflow
In GitHub Actions, an automated process defined in a YAML file. Triggered by events like pushing code or opening a PR.
Webhook
An automatic notification sent from one service to another when something happens. "When a form is submitted, send the data to my API."
Cron Job
A scheduled task that runs automatically at set times. Like an alarm clock for commands. "Run my morning briefing at 7 AM every day."
n8n
An open-source workflow automation tool. Alternative to Zapier/Make. Connects services together with visual flows.
Deploy
Making your code available on a server for users to access. Moving from development to production.
Production (prod)
The live environment where real users interact with your application. Handle with care.
Staging
A copy of production used for final testing before deploying. Catches issues before real users see them.
Vercel
A platform for deploying websites directly from GitHub. Community standard for simple deployments.
Railway
A cloud platform for deploying full applications (backend + database). Simple alternative to AWS.
Headless
Running software without a visible interface. "Headless Chrome" = a browser in the background. Used for automation and testing.
Log
A record of what a program did. When something breaks, check the logs. tail -f app.log watches a log in real-time.
Secrets
Sensitive values (API keys, passwords, tokens) that should never appear in code or git. Stored in environment variables or a secrets manager.
YAML
"YAML Ain't Markup Language." A human-readable config format. Used in Docker Compose, GitHub Actions, and many config files. Indentation matters.
JSON
JavaScript Object Notation. A data format used everywhere — APIs, config files, MCP settings. Looks like {"key": "value"}.
Makefile
A file containing shortcuts for common commands. make build or make deploy can run complex multi-step processes with a single word.
Linting
Automatically checking code for style issues, errors, and bad practices. A linter is like a spell-checker for code.
Hot Reload
When your application automatically restarts or updates as you save changes. Speeds up development.
Healthcheck
An endpoint confirming a service is running correctly. Often a URL like /health that returns "OK." Monitoring systems poll these.
Rollback
Reverting a deployment to a previous working version when something goes wrong. Your safety net for bad deploys.
Infrastructure as Code (IaC)
Managing servers through config files (Terraform, Docker Compose) rather than clicking dashboards. Reproducible and version-controlled.
SSH Tunnel
A secure connection forwarding traffic from your local machine through SSH to a remote server. Access private VPS services safely.
Monorepo
A single repository containing multiple projects or services. Your AI Second Brain repo is a monorepo — memory, config, and workflows in one place.

Networking & Security

HTTP / HTTPS
The protocol web browsers use to load pages. HTTPS = encrypted (secure). Always use HTTPS for anything sensitive.
REST API
A standard way to build web APIs using HTTP methods: GET = read, POST = create, PUT = update, DELETE = remove.
SSL / TLS
The encryption technology behind HTTPS. Ensures nobody can read data traveling between your browser and a server. Let's Encrypt provides free certificates.
VPN
Virtual Private Network. Creates a secure tunnel between your device and another network. Tailscale is a modern VPN for connecting your devices.
RLS
Row Level Security. A database feature (used in Supabase) that controls which users can see which rows of data.
Token (auth)
A temporary password that proves your identity to a service. Different from AI tokens — auth tokens are for security.
OWASP
Open Web Application Security Project. Publishes the "Top 10" list of web security risks. Their LLM Top 10 covers AI-specific threats.

Gen AI Circle Community

Second Brain
Your AI-enhanced knowledge management system built on Claude Code + Git + markdown files. A compounding knowledge repository.
claude-squad
A tool for managing multiple AI coding agents in parallel tmux sessions. Each agent gets its own Git worktree. Install via brew install claude-squad.
dmux
Similar to claude-squad but supports 12+ different AI agents (Claude, Codex, Gemini, etc.). Press n for new task, m to merge.
Boris Pattern
Running 5-10 separate Git checkouts of the same repo to run multiple Claude Code agents in parallel. Named after Boris, creator of Claude Code.
OpenClaw
An open-source autonomous AI agent platform. Can run 24/7 on a VPS. The community discovered 341 malicious skills targeting it — be careful with trust.
Superpowers Plugin
A Claude Code plugin with sophisticated planning and execution workflows. Recommended for structured project work.
Compound Engineering
A plugin/skill that optimizes how Claude Code plans and executes. Warning: can use 19K tokens just loading up.
Sidecar
A unified development workflow shell managing agent sessions, Git worktrees, and conversation history across all your AI tools.
COG Pattern
Claude Code + Obsidian + Git. A proven architecture where Claude operates on the same markdown files you browse in Obsidian.
Dual-Write Pattern
Saving data in two places simultaneously (e.g., markdown + Supabase) for redundancy and different access patterns.
Vibe Coding
Using AI to write code by describing what you want in natural language rather than writing code yourself.
YOLO Mode
Running Claude Code with full permissions on your main machine without security precautions. Not recommended — use containers instead.
Context Budget
How much context window is already consumed by MCPs, skills, and system instructions before you start typing. Fewer MCPs = more room for work.
mcp-builder
A skill that helps you create your own MCP servers. Used to build custom integrations for services like GoHighLevel.
mcpsec
A security scanning tool for MCP servers. Checks for known vulnerabilities in your installed MCPs.
Version Pinning
Locking an MCP or package to a specific version so it doesn't auto-update to a potentially broken or compromised version.
Remote Control
Official Claude Code feature (Feb 2026) that lets you start a task on your laptop and continue from your phone. Requires Max plan.
